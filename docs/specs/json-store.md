# Spec: JsonStore

## 개요
- JSON 파일을 읽고 쓰는 계층. JsonParser/JsonWriter를 파일
  입출력과 연결한다.

## 입력
- 파일 경로(`std::string`).
- 쓰기 시에는 파일 경로 + 저장할 `JsonValue`.

## 출력
- 읽기: 파일 내용을 파싱한 `JsonValue`.
- 쓰기: 성공 시 파일에 JSON 문자열이 저장됨 (반환값 없음, 실패
  시 예외).

## 엣지 케이스 / 예외 상황
- 파일이 존재하지 않으면 읽기 시 빈 배열(`JsonValue` array)로
  간주하고 예외를 던지지 않는다 (CLAUDE.md 에러 처리 원칙).
- 파일은 존재하지만 내용이 비어있으면 빈 배열로 간주한다.
- 파일 내용이 JSON 문법에 맞지 않으면 읽기 시 예외를 던진다.
- 쓰기 시 디렉터리가 없는 등 파일을 열 수 없으면 예외를 던진다.
- 쓰기는 기존 파일 내용을 덮어쓴다.

## 테스트 케이스 목록
- [ ] 정상 케이스: JsonValue를 파일에 쓰고 다시 읽으면 동일한
      값이 나온다 (round-trip)
- [ ] 정상 케이스: 이미 존재하는 파일을 다시 쓰면 이전 내용을
      덮어쓴다
- [ ] 엣지 케이스: 존재하지 않는 파일을 읽으면 빈 배열 반환
- [ ] 엣지 케이스: 내용이 빈 파일을 읽으면 빈 배열 반환
- [ ] 예외 케이스: 문법이 잘못된 파일을 읽으면 예외 발생
