# Spec: Create

## 개요
- 새 연락처(Record)를 입력받아 JSON 파일에 추가하는 기능.
- `id`는 사용자 입력을 받지 않고 프로그램이 자동으로 생성한다
  (CLAUDE.md 데이터 모델 원칙).

## 입력
- 파일 경로 (`std::string`).
- `name`, `phone` (`std::string`).

## 출력
- 생성된 `Record` (자동 생성된 `id` 포함).
- 파일에는 기존 레코드 목록 + 새 레코드가 저장된다.

## 엣지 케이스 / 예외 상황
- 파일이 없거나 비어있으면 (JsonStore가 빈 배열로 처리) 첫 레코드의
  `id`는 1이 된다.
- `id`는 "기존 레코드 중 최댓값 + 1"로 생성한다 (개수 기준이 아님).
  삭제로 인해 중간 id가 비어도 새 id가 기존 id와 겹치지 않게 하기
  위함.
- `name`, `phone`이 빈 문자열이어도 그대로 저장한다 (입력값 검증은
  이 기능의 책임이 아님).

## 테스트 케이스 목록
- [x] 정상 케이스: 빈 파일에 첫 레코드 추가 시 id는 1
- [x] 정상 케이스: 레코드가 있는 파일에 추가 시 id는 기존 최댓값 + 1
- [x] 정상 케이스: 추가 후 파일을 다시 읽으면 새 레코드가 포함됨
- [x] 정상 케이스: 기존 레코드는 그대로 유지됨 (덮어쓰지 않음)
- [x] 엣지 케이스: 중간 id가 삭제되어 비어 있어도 최댓값 + 1로 생성
- [x] 엣지 케이스: 따옴표/백슬래시/한글이 섞인 이름도 파일 저장 후
      다시 읽으면 원래 값 그대로 유지
