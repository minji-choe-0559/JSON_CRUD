# 커밋 컨벤션

## 줄바꿈 규칙
- 제목, 본문 모든 줄은 55자를 넘지 않도록 줄바꿈.
- 55자를 넘는 문장은 의미가 자연스럽게 끊기는 지점에서 줄바꿈.

## 제목
- 형식: `[category] Title`
  - 예: `[feat] Enable json read/write`
- 카테고리 종류:
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `refactor`: 동작 변화 없는 코드 구조 개선
  - `docs`: 문서(md 등) 변경
  - `test`: 테스트 코드 추가/수정
  - `chore`: 빌드 설정, 의존성 등 잡무성 변경
  - `style`: 포맷팅 등 코드 의미에 영향 없는 변경

## 본문
- 한국어로 작성하되, 영어 단어가 더 자연스러우면 번역하지 않고 그대로 사용.
- 추가된 기능, 수정된 버그, 리팩토링 이유/결과 등을 항목별로 나열.
- 각 항목은 `-`로 시작.
- 각 문장은 명사형으로 종결 (예: "~함", "~추가" 대신 "~추가함"이 아니라 "~추가" 형태).
- 문장은 간결하게, 이해하기 쉽게 작성.
- 한 항목이 55자를 넘으면 다음 줄로 이어서 씀 (이어지는 줄은 `-` 없이 들여쓰기).

## 예시
```
[feat] Enable json read/write

- nlohmann/json 기반 JsonStore 클래스 추가
- data.json 파일 read/write 함수 구현
- 파일 없을 시 빈 배열로 초기화하는
  예외 처리 추가
```

```
[fix] Handle missing id on update

- 존재하지 않는 id로 update 시도 시 크래시 나던 버그 수정
- 존재하지 않는 id 조회 시 안내 메시지 출력으로 변경
```

```
[refactor] Split JsonStore from main

- main.cpp에 섞여있던 JSON read/write 로직 분리
- 메뉴 루프와 데이터 처리 로직 간 결합도 감소
```
